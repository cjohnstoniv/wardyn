died with the branch. This is the durable record: what was measured, how, what it
found, and what is still open.

## How the score works

`score = min(raw, ceiling)` per dimension; composite = weighted mean (security and
reliability count double; Σw = 14).

- **raw** = `(pass + 0.5·partial) / (pass + partial + fail)` over that dimension's
  criteria verdicts.
- **ceiling** = the worst OPEN finding above low-confidence in that dimension:
  crit→35, high→60, med→85, none→100.
- Any open high-confidence HIGH clamps the whole composite to ≤74.

Two consequences worth knowing before reading any movement in these numbers:

1. **Clearing findings is worthless where raw < ceiling.** Mid-campaign, 10 of 12
   dimensions were raw-bound, so a full MED burn-down would have bought zero
   points. It was descoped for that reason and the reasoning recorded, rather than
   run to look busy.
2. **Coverage (`cov`) never enters the score.** It is a trust signal only. Security
   sits at 55% — meaning nearly half its criteria are unevaluated, so treat its 82
   as the least-settled number on the board.

## What makes a claim count

- **Resolution is machine-verified.** A finding is only credited when its
  `resolved:{sha,test}` re-proves on every scorer run: the commit must be an
  ancestor of the scored ref *and* the named test must still exist in the tree. An
  unbacked marker is silently ignored — it can under-count, never fake. (A
  simulated marker with a fabricated sha was, correctly, ignored.)
- **Verdicts move only by blind re-assessment.** Reviewers read the current code
  with no prior verdicts and no target score. Hand-editing a verdict is forbidden.
- **Fixes carry counterfactual tests.** Each fix below was verified by reverting it
  and watching the test fail. Where a test could not be a true counterfactual
  (e.g. a pin guarding against a future regression), it is labelled as a pin.

## What the completed review found

Five "crown jewel" lanes had never been adversarially deep-reviewed (an earlier
spend cap). Completing them cost 5 points on the spot: **207 surviving findings**
(27 more were refuted and dropped), including **19 real HIGHs** after applying the
refuters' own "real but overstated" downgrades. All 19 are now fixed:

| Defect | Evidence |
|---|---|
| Concurrent runs shared one egress allowlist — `resolvePolicy` handed out a shallow copy of the process-global default policy, so two runs appended into the same backing array | Reproduced under `-race`: a run loses its own domain and inherits a sibling's |
| The managed-harness login run recorded a live ~1-year OAuth token verbatim into a replayable asciicast | Token appears in the persisted cast with the gate disabled |
| The 64 MiB recording-upload cap was dead — `MaskingWriter.Close` closed a borrowed pipe, so `io.Pipe`'s once-only error store swallowed every copy error | Over-cap upload returned 204 + audited "success" while truncating |
| `SetWorkspaceImportState` was the last unfenced workspace writer; the C001 guard was only a read | A finalize decided from a stale read applies over a live run's claim |
| The per-run JWT-SVID had a 1h TTL and no refresh producer; the token lives in the proxy sidecar, so at expiry *every* consumer died at once — including the injector's OAuth re-resolve | Removing the renew route 404s the whole path |
| A second published age key was accepted by the boot guard (a single-value denylist) and was the committed default of two dev scripts | Guard now refuses a set; scripts mint ephemeral keys |
| BYOI "wrap-only" was defeated by ONBUILD triggers (host-side RCE at wrap time) | Now refused; the inspect→build TOCTOU is closed |

## Honest residuals

- **Security coverage is 55%.** Two lanes still carry unevaluated criteria. The 82
  is the least-settled score here.
- **Function-level complexity is not bounded.** The `runs.go` decomposition
  (2355→585) is real and provably pure code motion, but it redistributed lines
  without breaking up the god functions: seven functions still exceed 200 lines,
  the largest being `cmd/wardynd`'s `run()` at 676 — bigger than the
  `dispatchWithVerify` (601) everyone was watching. **Correction to the record:** commit `66d8faa`'s message claims
  this function went "~697 → ~546". The true measurement is **704 → 599** (601 at
  HEAD) — a ~105-line shrink. The commit message repeated an unverified figure; the
  blind reviewer caught it. The claim was wrong, the decomposition was not.
- **Two hand-maintained HTTP clients remain** (`pkg/client` vs `cmd/wardyn/client.go`) —
  duplicate `do`, ~15 line-for-line duplicated methods, and a `policyBody` twin that is
  the same shape that let the create-run DTO drift in the first place. That DTO is now
  down to two declarations pinned by a structural parity test (a blind reviewer rightly
  noted "single-sourced" overstates it); the real fix is a type alias.
- **Env-var validation is real but partial.** The typed helpers now exit 2 naming the
  variable and value instead of silently swallowing bad input — but only 19 of 57 call
  sites are typed, and `cmd/wardyn-git-helper` still hand-parses a duration outside
  `cliutil` entirely, reinstating its default on a typo: the exact bug, still live in
  one place. 112 distinct `WARDYN_*` vars exist with no central registry.
- **List endpoints have no pagination contract**; orchestrator ref→substrate
  routing is still process-memory-only (the concrete blocker for a K8s driver).
- **The MITM CA private key transits a plaintext container env var**
  (`WARDYN_PROXY_CONFIG_JSON`), and the proxy sidecar runs on the daemon-default
  runtime regardless of the agent's tier.
- **`GO-2026-5932`** (x/crypto/openpgp, unmaintained, no fix) is accepted: reached
  only via `filippo.io/age`, and the flagged subpackage is never imported.
  `govulncheck` reports 0 called-path vulnerabilities on both tag sets.
- **11 `log.Printf` sites remain** in `internal/api` and `internal/egress/proxy` — both
  100% unstructured, including the control-plane API package, and the slog guard test
  explicitly excludes them, so no ratchet protects that surface.
- **Deferred for want of hardware**: live eBPF cross-container recovery and the
  tetragon export-scope work need a privileged host.

## Reproducing this

```bash
SCORED_REF=HEAD node local/review-2026-07-16/scratch/gen-report.js
```

Recomputes every number from the unchanged formula, re-proving each resolution
marker from git. The scratch tree is gitignored by design — this document is the
part meant to survive.
